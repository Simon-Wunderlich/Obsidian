# Design decisions


| Design Issue       | Architectural Style                                                                                                                                                                                                         |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| ------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Context**        | The system is a turn by turn bus navigation system which will often be able to rely on consistent data but will occasionally communicate with other buses and administrators to receive updates to the routes and schedules |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| Quality attributes | Scalability, performance                                                                                                                                                                                                    |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| **Option 1**       | Identifier                                                                                                                                                                                                                  | Service based                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
|                    | Description                                                                                                                                                                                                                 | Clients interact with a centralised database abstracted via services which can be accessed via an API gateway                                                                                                                                                                                                                                                                                                                                                                                                                                             |
|                    | Status                                                                                                                                                                                                                      | Accepted                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
|                    | Evaluation                                                                                                                                                                                                                  | Scalability: Very scalable as each client is able to communicate and transmit data to each other via the database.<br>Performance: Less performant as each query and request is throttled by network speed.                                                                                                                                                                                                                                                                                                                                               |
|                    | Rationale                                                                                                                                                                                                                   | As each bus is required to have access to real time information about position and road conditions from other buses, using a Service based architecture is the best choice. This is because it allows each client to upload and recieve data from a database which is being updated by all other clients in real time. While performance is an issue, especially in rural areas where network connectivity is not guaranteed, this could be offset by caching all necessary data for each request.                                                        |
|                    |                                                                                                                                                                                                                             |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| **Option 2**       | Identifier                                                                                                                                                                                                                  | Layered                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
|                    | Description                                                                                                                                                                                                                 | Each client has all logic and data stored and run locally on each device                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
|                    | Status                                                                                                                                                                                                                      | Rejected                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
|                    | Evaluation                                                                                                                                                                                                                  | **Scalability**: As each user is it's own standalone system, all functionality which does not require communication between buses will work very well and at any scale.<br><br>**Performance**: As each bus will have it's own local database,  queries will not be throttled by network speeds and will have consistently fast data retrieval speeds                                                                                                                                                                                                     |
|                    | Rationale                                                                                                                                                                                                                   | While having a local database greatly improves performance and robustness, the Layered architecture fails in a system which requires users to be able to share and receive real time data from each other. Functional requirements such as hazard reporting will not be possible if each user is self contained. Additionally it is significantly more difficult to alter routes and schedules as an Administrator if the system only relies on a local database for this information as they would need to modify each device individually and manually. |
